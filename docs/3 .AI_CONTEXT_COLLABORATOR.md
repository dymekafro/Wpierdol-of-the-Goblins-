## 1. Elevator pitch + core loop

**Wpierdol of the Goblins** — open-world action RPG in one continuous **magic dark forest** map. Player is a **druid**, last guardian of the woods, with a **home base (Groves / Sady)** at map center.

**Core loop:**

```
Druid base (save, heal, craft stub) → explore forest → find goblin camp
→ fight (totem + goblins) → camp Cleared → return optional
→ capture camp (Captured) → permanent bonus → next camp (farther = harder rings)
```

**Not level-based.** Single world scene `World.unity`. Difficulty scales by **distance rings** from base, not level numbers.

**Tone:** dark magical forest, goblins are real threat; title humor is situational.

## 3. Technical architecture

### Bootstrap pattern

Scenes are **minimal**: one GameObject `Bootstrap` + one MonoBehaviour that **procedurally builds** everything at runtime.

| Scene | Bootstrap script | Builds |
|-------|------------------|--------|
| `MainMenu` | `MainMenuBootstrap` | Camera, canvas, buttons, wires `GameManager` |
| `CharacterCreation` | `CharacterCreationBootstrap` | Attribute allocation UI → `GameManager.attributes` |
| `World` | `WorldBootstrap` | `WorldGenerator`, player, camera, HUD, pause |

**Do not** expect pre-placed player/camps in World scene hierarchy (except optional `Atmosphere` for saved fog). World content is created in `WorldGenerator.Generate()`.

### Singletons (DontDestroyOnLoad or static)

| Type | Role |
|------|------|
| `GameManager` | Cross-scene state: attributes, campStates, pendingLoadData, isContinuing |
| `SettingsManager` | Volume, mouse sensitivity JSON |
| `GameAudioManager` | SFX play helpers |
| `SaveSystem` | Static JSON save/load |
| `GameAssetRegistry` | Static prefab/audio resolution + cache |
| `ForestAtmosphereSettings` | Optional scene singleton for fog/sun (Instance) |

### Player drivers (dual path)

`WorldBootstrap.useInvectorLocomotion` **default true**:

- **Primary:** `InvectorPlayerBuilder.TryBuild()` → prefab `ThirdPersonController_LITE` + `vThirdPersonCamera_LITE` + WPG layer (`PlayerStats`, `PlayerCombat`, `InteractionDetector`, `InvectorPlayerAdapter`, `CharacterAnimDriver`).
- **Fallback:** if prefab missing → `PlayerBuilder.BuildDruid()` + `ThirdPersonCamera` + GanzSe model.

**Input split:**

- Invector: **Old Input Manager** (`Input.GetAxis`, WASD, mouse).
- WPG combat/UI: **New Input System** (`Keyboard.current`, `Mouse.current`).
- **Never** set Active Input Handling to Input System Package only.

### Damage pipeline

`IDamageReceiver` (`WPG.Core`) implemented by `PlayerStats`, `GoblinBase`, `Totem`. Combat uses `Physics.OverlapSphere` + `GetComponentInParent<IDamageReceiver>()`.

### Materials

Runtime materials via `MaterialFactory.Get()` and `MaterialUpgrader.UpgradeHierarchy()` — avoids pink missing-shader on URP.

## 4. Scene flow + Build Settings

**Scene name constants:** `WPG.Core.SceneNames`

```csharp
MainMenu = "MainMenu"
CharacterCreation = "CharacterCreation"
World = "World"
```

**EditorBuildSettings order (actual repo):**

1. `Assets/_Game/Scenes/MainMenu.unity`
2. `Assets/Invector-3rdPersonController_LITE/3D Models/Characters/Invector@V-Bot 2.0/Prefab/CharacterCreation.unity`  
   (contains `CharacterCreationBootstrap` on Bootstrap — WPG UI, not Invector demo flow)
3. `Assets/_Game/Scenes/World.unity`

**Transitions:**

- `GameManager.StartNewGame()` → clear state → `LoadScene(CharacterCreation)`
- `CharacterCreationBootstrap` confirm → `GoToWorldFromCreation()` → `LoadScene(World)`
- `GameManager.ContinueGame()` → `SaveSystem.Load()` → fill `pendingLoadData`, `campStates` → `LoadScene(World)`
- `GoToMainMenu()` from pause → `LoadScene(MainMenu)`

**World spawn:** `WorldBootstrap` uses `WorldGenerator.SpawnPoint` (druid base) unless continue save has `hasPosition`.

## 6. Systems detail

### Combat

| Action | Input | Code | Notes |
|--------|-------|------|-------|
| Melee | LPM | `PlayerCombat.DoMelee()` | Damage `5 + STR*2`; arc overlap |
| Fireball | E | `DoFireball()` | Mana 15; dmg `SpellPower * 4`; aims camera forward if no `PlayerController` |
| Heal | Q | `DoHeal()` | Mana 25; heal 35 HP |
| Interact | E/F near camp | `InteractionDetector` + `CampInteractable` | E consumed by interact if `HasReadyInteractable` |

Goblin melee damage 5; archer arrow 8. Totem 80 HP.

### Save

- File: `wpg_save.json` via `SaveSystem.SavePath`
- Written in `DruidBase.TrySave()` on enter + periodic cooldown
- Fields: position, HP, mana, `PlayerAttributes`, `camps[]` with `campId` + `CampState`, `visitedPowerSites`
- Menu Continue uses `GameManager.ContinueGame()`

**Do not change** `Save()`, `Load()`, `HasSave()` signatures without team agreement.

### Camps (runtime IDs from WorldGenerator)

| campId | displayName | ring | MVP composition |
|--------|-------------|------|-----------------|
| `goblin_camp_first_clearing` | Pierwsza Zagroda | 1 | 2 storm, 1 archer |
| `goblin_camp_forest_den` | Leśna Nora | 1 | 3 storm |
| `goblin_camp_ember_moss` | Mchy Żaru | 2 | 3 storm, 2 archer |
| `goblin_camp_lost_roots` | Korzenie Zagubione | 2 | 2 storm, 3 archer |
| `goblin_camp_shade_glade` | Cienista Polana | 3 | 4 storm, 3 archer, 2 elite |

`GoblinCamp.CheckCleared()` when totem dead AND all goblins dead → `Cleared`.  
`TryCapture()` only from `Cleared` → `Captured` + bonus by `captureBonusType`.

`GameManager.campStates` synced on state change; restored on Continue.

### GameAssetRegistry

- Initializes on play; logs `[GameAssetRegistry] === Imported Packs ===` and slot resolution
- Primary paths often `Assets/_Game/Prefabs/...` (folder mostly empty — **expected**)
- Falls back to Asset Store folders + fuzzy token search in `Assets/`
- Editor: **WPG → Refresh Asset Registry**

See `docs/INVECTOR_INTEGRATION.md`.

Key facts:

- Prefab: `GameAssetPaths.InvectorThirdPersonPrefab` → `ThirdPersonController_LITE`
- Camera: `vThirdPersonCamera_LITE`
- `WorldBootstrap.useInvectorLocomotion = false` → full legacy path

## 12. Example prompts for AI assistant

Use these with this context loaded:

**New camp**

> Dodaj szósty obóz goblinów w `WorldGenerator.BuildCamps()`: campId `goblin_camp_moss_bridge`, nazwa PL „Most Mszy”, pierścień 2, 50m na kącie 160°, 2 szturmowców i 2 łuczników. Nie zmieniaj SaveSystem ani sygnatury GoblinCamp.ApplyState.

**Fireball damage**

> Zwiększ obrażenia fireballa o ~20% zmieniając tylko `PlayerCombat.DoFireball()` lub formułę SpellPower w `PlayerAttributes` — pokaż diff minimalny.

**Fog preset**

> Ustaw domyślne wartości w `ForestAtmosphereSettings` na chłodniejszą niebiesko-zieloną mgłę (fogDensity ~0.022) i opisz jak zapisać to w World.unity po Play.

**Goblin HP**

> W `GoblinStormtrooper.Awake()` ustaw maxHealth na 45 i dodaj jedno zdanie w komentarzu po polsku dla designerów.

**Invector off**

> Jak tymczasowo wrócić do legacy GanzSe gracza? Które pole w WorldBootstrap i jakie skutki dla kamery?

**Import pack**

> Po imporcie „Basic RPG Sounds” gdzie sprawdzić czy registry widzi SfxHit i jakie ścieżki dodać w GameAssetPaths jeśli folder ma inną nazwę?

**Safe save field**

> Dodaj do SaveData pole `playTimeSeconds` i zapisuj je w `GameManager.BuildSaveData` — bez zmiany sygnatury SaveSystem.Save/Load.
